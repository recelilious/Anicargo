# Anicargo 阶段进度

更新时间：2026-03-22

## 当前已完成

- Git 仓库、目录结构、`.gitignore` 与基础文档已经初始化完成。
- Web 前端的第一阶段交互与视觉基线已经确定。
- 访客设备身份、可选用户登录、独立管理员入口的基础身份模型已经接通。
- Bangumi 元数据接入、Yuc 放送时间补全、番剧详情与搜索页的主要交互已经落地。
- 后端已经具备 SQLite、订阅阈值配置、字幕组规则管理、管理员基础面板等核心基础设施。
- Bangumi 上游错误处理已经做了第一轮加固，日历接口具备失败回退缓存。

## 当前已确认的技术方向

- 元数据：Bangumi API
- 放送时间补全：Yuc 页面解析
- Web：React + TypeScript + Fluent 2 风格
- 后端：Rust + Axum + SQLite
- 部署优先级：Linux 优先，Windows 兼容

## 现在还未完成的核心部分

- 下载引擎本体
- 订阅阈值与下载任务的正式联动
- 资源搜索、候选资源筛选、字幕组优先级与替换窗口执行
- 实际的剧集入库、文件管理、播放资源映射
- 管理员对下载任务的强制触发、观察与干预能力

## 下一阶段目标

这一阶段开始正式进入下载链路：

- 建立下载任务、资源、状态流转的数据模型
- 将订阅变化与下载调度联动起来
- 抽象下载引擎接口，为后续接入真实 torrent 引擎做准备
- 先完成“可调度、可追踪、可扩展”的后端骨架，再逐步补齐实际下载执行
